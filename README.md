# 🛠️ Sub-Tool

A lightweight, cross-platform subscription converter for **Sing-box**, written in Go.

## 🌟 Key Features

- **Protocol Support**: Seamlessly parses VMess, VLESS, Trojan, Shadowsocks, and Hysteria2.
- **Template-Driven**: Total control over the output configuration via `template.json`.
- **Preserved Order**: Maintains the logical order of fields (Log -> DNS -> Outbounds -> Route) as defined in your template.
- **Smart Grouping**:
    - **Auto-Region Groups**: Automatically categorizes nodes into country-based groups using Regex.
    - **Dynamic Fallback**: Detects the primary tag from your template (e.g., "🎯 Direct") and uses it as a fallback for empty region groups.
- **Placeholder System**:
    - `<all-proxies>`: Expands to all filtered physical node tags.
    - `<all-region-groups>`: Expands to all generated region group tags.
    - `<dynamic-region-groups>`: Injects dynamically generated `urltest` region groups.
- **CLI Enhanced**: Features ANSI color output, strict argument validation, and detailed help info.

## 🚀 Quick Start

### 1. Build
Build the project using the provided `Makefile`:

```bash
# Build local dev version
make build

# Build for all platforms (Linux, Windows, macOS, Android)
make release
```

### 2. Configure Template
Create a `template.json` file, and update the `_extra` field in your `template.json`, for example:

```json
{
  "_extra": {
    "sub_url": "YOUR_SUBSCRIPTION_URL",
    "filter_keywords": "Reset|Remaining|Expired",
    "regions": [
      ["Hong Kong|HK", "🇭🇰 Hong Kong"],
      ["United States|US", "🇺🇸 United States"]
    ]
  },
  "log": { ... },
  "outbounds": [ ... ]
}
```

### 3. Usage
```bash
# Run using URL from template
./sub-tool

# Manually specify URL and output path
./sub-tool -url "http://example.com/sub" -output my_config.json

# Show help and available placeholders
./sub-tool -h
```

## 🛠️ Placeholder Reference

| Placeholder | Description |
| :--- | :--- |
| `<all-proxies>` | Used inside `outbounds` arrays. Expands to all filtered physical nodes. |
| `<all-region-groups>` | Expands to all tags of groups generated by `_extra.regions`. |
| `<dynamic-region-groups>`| Injects the actual group objects into the `outbounds` list. |
